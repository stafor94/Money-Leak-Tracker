# 고정비 누수 탐지기

**고정비 누수 탐지기**는 매달 반복해서 나가는 구독료와 고정비를 한곳에서 관리하는 모바일 우선 개인 금융 웹앱입니다. HTML, CSS, 바닐라 JavaScript만 사용하며 백엔드나 빌드 과정 없이 깃허브 Pages에 바로 배포할 수 있습니다.

## 주요 기능

- 월 고정비, 연간 고정비, 활성 항목 수, 다음 결제, 잠재 절약액을 보여주는 대시보드
- 고정비 추가, 수정, 삭제, 활성/일시정지/해지 상태 변경
- 월간, 연간, 주간, 3개월, 6개월, 커스텀 청구 주기의 월/연 비용 정규화 계산
- 이름 또는 메모 검색, 카테고리/상태 필터, 다양한 정렬 옵션
- 이번 달, 7일 이내, 30일 이내, 이미 지난 결제일 보기
- 미사용, 낮은 만족도, 비필수 항목을 빠르게 선택할 수 있는 절약 시뮬레이터
- 사용량이 낮거나 없고 만족도가 낮으며 필수가 아닌 활성 항목을 해지 후보로 자동 표시
- 다크 모드, 통화 설정, JSON 내보내기/가져오기, 초기화 확인
- 설치형 앱처럼 보이도록 기본 PWA 매니페스트 제공

## 로컬에서 실행하기

브라우저에서 `index.html` 파일을 직접 열어도 됩니다. 또는 아래 명령으로 정적 서버를 실행할 수 있습니다.

```bash
python3 -m http.server 8000
```

그 다음 브라우저에서 `http://localhost:8000`으로 접속하세요.

## 깃허브 Pages 배포 방법

1. `index.html`, `style.css`, `app.js`, `manifest.json`, `README.md` 파일을 저장소에 커밋합니다.
2. 브랜치를 GitHub에 푸시합니다.
3. 저장소의 **Settings → Pages** 메뉴를 엽니다.
4. 배포할 브랜치와 루트 폴더를 선택합니다.
5. 저장 후 생성된 깃허브 Pages 주소로 접속합니다.

## 캐시 갱신 방식

`index.html`은 CSS, JavaScript, 매니페스트를 버전 쿼리(`?v=YYYYMMDD`)와 함께 불러옵니다. 배포 후 변경사항이 즉시 반영되도록 정적 파일을 수정할 때는 `index.html`의 `style.css`, `app.js`, `manifest.json` 뒤 버전 값을 함께 올려 브라우저가 이전 캐시 대신 새 파일을 요청하게 하세요.

## 데이터 저장 방식

모든 데이터는 사용자의 브라우저 `localStorage`에 저장됩니다. 서버, 로그인, 외부 데이터베이스, 네트워크 동기화는 없습니다. 브라우저나 기기를 옮길 때는 **JSON 내보내기**로 백업한 뒤 **JSON 가져오기**로 복원하세요.

## 제한 사항

- 데이터는 현재 브라우저와 기기에만 저장됩니다.
- 자동 알림이나 푸시 알림은 제공하지 않습니다.
- 은행, 카드사, 결제 서비스와 자동으로 연동하지 않습니다.
- 결제 예정일은 각 항목의 결제일을 기준으로 계산하며 실제 결제사의 세부 청구 일정과 다를 수 있습니다.

## 향후 개선 아이디어

- 오프라인 캐싱을 위한 서비스 워커 추가
- 결제 전 알림 기능
- 카테고리별 월별 추이 차트
- CSV 가져오기/내보내기
- 선택형 클라우드 동기화
